Justifique essa escolha considerando:

Alternativa 1:
WEBFLOW
Prós:
- manutenção mais simples para equipe não técnica
- bom para páginas institucionais
- editor visual amigável
Contras:
- interações premium mais customizadas podem ficar limitadas
- risco maior de visual com cara de builder se mal executado
- portfolio interativo avançado pode exigir workarounds

Alternativa 2:
FRAMER
Prós:
- rapidez de execução
- motion e prototipagem muito boas
- boa opção para landing pages
Contras:
- menor robustez para personalizações mais complexas
- menor controle estrutural que Next.js
- pode ficar menos flexível para futuras integrações e evolução

Você deve também mencionar que WordPress + Bricks pode ser funcional, mas não é a recomendação ideal para este caso devido à necessidade de refinamento visual premium, performance de alto nível e experiência altamente autoral.

Comandos exatos que você deve seguir no desenvolvimento:
- Gere o código completo em Next.js 15 com App Router
- Use React + TypeScript
- Use Server Components sempre que fizer sentido
- Use Server Actions quando aplicável
- Use Tailwind CSS com design system bem definido
- Use Framer Motion para animações sutis
- Organize o projeto em estrutura limpa e pronta para deploy na Vercel
- Gere componentes reutilizáveis
- Estruture tokens de design e componentes base antes de montar a página
- Entregue tudo pronto para produção
